# Многопоточность (Threading) в Python

**Многопоточность (multithreading)** — это способ выполнения программы, при котором несколько частей кода (потоков) работают одновременно внутри одного процесса.

---

# Основные понятия

- **Процесс** — запущенная программа
- **Поток (thread)** — отдельная линия выполнения внутри процесса

Один процесс может содержать несколько потоков, которые делят общую память.

---

# Зачем нужна многопоточность

## Основные сценарии

- Работа с сетью (API, HTTP-запросы)
- Чтение и запись файлов
- Обработка нескольких пользователей (серверы)
- Поддержание отзывчивости GUI

Позволяет не «ждать» завершения одной задачи, а выполнять другие.

---

# Особенность Python — GIL

**GIL (Global Interpreter Lock)** — это механизм, который позволяет выполнять только один поток Python-кода одновременно.

## Следствия

Хорошо подходит для I/O задач  
Плохо подходит для CPU-вычислений

---

# Использование threading

## Пример

```python
import threading
import time

def task(name):
    for i in range(3):
        print(f"{name}: {i}")
        time.sleep(1)

# создаём потоки
t1 = threading.Thread(target=task, args=("Поток 1",))
t2 = threading.Thread(target=task, args=("Поток 2",))

# запускаем
t1.start()
t2.start()

# ждём завершения
t1.join()
t2.join()

print("Готово")
```

---

# Как это работает

- `Thread` — создаёт поток
- `start()` — запускает выполнение
- `join()` — блокирует основной поток до завершения

Потоки выполняются конкурентно (перемешанно)

---

# Проблемы многопоточности

## Гонка данных (Race Condition)

Возникает, когда несколько потоков одновременно изменяют общие данные.

```python
counter = 0

def increment():
    global counter
    for _ in range(100000):
        counter += 1
```

Результат будет непредсказуемым

---

# Синхронизация потоков

## Lock (блокировка)

```python
lock = threading.Lock()

def increment():
    global counter
    for _ in range(100000):
        with lock:
            counter += 1
```

Гарантирует, что только один поток работает с ресурсом

---

# Дополнительные инструменты

- `Lock` — базовая блокировка
- `RLock` — рекурсивная блокировка
- `Semaphore` — ограничение количества потоков
- `Event` — сигнал между потоками
- `Queue` — потокобезопасный обмен данными

---

# Когда использовать

## Подходит

- I/O операции
- Сетевые приложения
- GUI
- Параллельная обработка задач

## Не подходит

- Тяжёлые вычисления (CPU-bound задачи)

Для этого лучше использовать multiprocessing

---

# Итог

Многопоточность — это инструмент для:

- одновременного выполнения задач
- повышения отзывчивости программы
- эффективной работы с I/O

Но из-за GIL не ускоряет вычисления.
